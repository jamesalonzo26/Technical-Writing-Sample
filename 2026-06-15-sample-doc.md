# Introduction
This is a sample documentation post created to demonstrate how technical writing can be published using **GitHub Pages** and **Markdown**.  
It shows the structure, clarity, and formatting that employers expect in a professional portfolio.

---

# Purpose
The goal of this post is to:
- Showcase basic Markdown formatting
- Demonstrate use of headings, lists, and emphasis
- Provide a real example of documentation hosted on GitHub Pages

---

# Example Section: Writing Guidelines
When creating documentation, keep these principles in mind:

1. **Clarity** – Write in simple, direct language.  
2. **Consistency** – Use standardized terminology across all documents.  
3. **Structure** – Organize content with headings, lists, and tables.  
4. **Accuracy** – Ensure technical details are correct and up to date.
